When a prior is “flat” that means it has no prior information at all. These priors need to be informed for the model to work.

prior.m2_beta = c(
  set_prior("normal(0, 2)", class = "Intercept"),
  set_prior("normal(0, 1)", class = "b"),
  set_prior("exponential(1)", class = "sd"),

  set_prior("exponential(1)", class = "phi"),
  # Added a prior for the Zero-Inflation probability
  set_prior("logistic(0, 1)", class = "zi") 
)

What do the priors mean? Centered at 0, fairly wide spread. This is a log scale, so 0-1 prior would allow 0.1 to 50 plants. Which is not going to occur, hence why it is weakly informative.

The Beta: Tells us how much cover there is if the plant is present.

The ZI: Tells us the probability that the plant is absent.

Weakly informative priors are set. Now let’s run the model. Setting priors is a bit of an art. It requires knowledge of the system and an informed opinion.

m2.brms=brm(formula = nValue_prop ~ SoilType + ESD_Designation + (1 | SiteID/Transect), data = veg, 
            family = zero_inflated_beta(),
            prior = prior.m2_beta, chains = 4, cores = 4,
